Overview

FiTest Extension is an internal QA automation tool developed by Ihsan Solusi Informatika. It enables authorized testers to record user interactions within the company's internal web application — the FITTEST platform — generating XPath-based test scripts by capturing clicks, inputs, and file upload events during an active recording session.

This extension is purpose-built and tightly coupled to the FITTEST application. It does not operate on, interact with, or collect data from any other website or web application. Attempting to use the extension on any page outside of the FITTEST platform will result in no functionality being available.

This extension is exclusively designed for internal use with the FITTEST platform and is inaccessible without a valid corporate VPN connection. It is not intended for general public use and cannot function as a general-purpose browser tool.

Data We Collect

The extension collects the following data only during an active recording session:

Mouse clicks & hover position

Captured to generate XPath selectors for interacted elements.

Keyboard input values

Captured on input, textarea, and select elements to record form interactions.

Authentication information

Login credentials entered into the internal web application may be captured as part of input recording. This data is transmitted only locally via postMessage to the Control window on the same device — never stored persistently or sent to external servers.

File upload events

File name, type, and base64 content captured for test replay purposes.

XPath selectors

Generated from DOM structure to uniquely identify interacted elements.

What We Do Not Collect

Personally identifiable information
Browsing history or external URLs
Financial or payment information
Location or health data
Personal communications
Full website content or page structure

How Data Is Used & Transmitted

All recorded interaction data is transmitted locally via the browser's postMessage API to the Test Recorder Control window running on the same device. No data is sent to any external server, third-party service, or cloud platform.

Data exists only within the active browser session and is used solely to compile reusable automated test scripts for internal QA purposes.

Permissions Explained

activeTab

Access the active tab to inject the recorder and capture user interactions during a test recording session.

scripting

Inject the test recorder script into the target tab when a recording session is initiated by the Control window.

storage

Persist recording session state (e.g. active/inactive) across tab navigations. No personal data is stored.

host_permissions

The FITTEST internal application may be served under varying internal hostnames depending on the team's VPN and network configuration. Broad host permission ensures the extension can activate regardless of which internal hostname is in use. The extension does not collect any data from public websites — it is functionally inactive on any page outside the FITTEST platform, as its recorder logic and UI controls are only initialized when communicating with the FITTEST application.

Scope of Operation

FiTest Extension is exclusively designed to work with the FITTEST platform — the company's internal QA automation application. It has no functionality outside of this context.

Active on

The FITTEST internal web application, accessible via authorized corporate VPN only.

Inactive on

Any public website, third-party web application, or any URL outside the FITTEST platform — no data is recorded or transmitted in these contexts.

The extension's recording logic, control UI, and data transmission are only initialized through a deliberate action within the FITTEST application. There is no passive background activity on other tabs or websites.

Overview

FiTest Extension is an internal QA automation tool developed by Ihsan Solusi Informatika. It enables authorized testers to record user interactions within the company's internal web application, generating XPath-based test scripts by capturing clicks, inputs, and file upload events during an active recording session.

This extension is exclusively designed for internal use and is inaccessible without a valid corporate VPN connection. It is not intended for general public use.

Data We Collect

The extension collects the following data only during an active recording session:

Mouse clicks & hover position

Captured to generate XPath selectors for interacted elements.

Keyboard input values

Captured on input, textarea, and select elements to record form interactions.

Authentication information

Login credentials entered into the internal web application may be captured as part of input recording. This data is transmitted only locally via postMessage to the Control window on the same device — never stored persistently or sent to external servers.

File upload events

File name, type, and base64 content captured for test replay purposes.

XPath selectors

Generated from DOM structure to uniquely identify interacted elements.

What We Do Not Collect

Personally identifiable information
Browsing history or external URLs
Financial or payment information
Location or health data
Personal communications
Full website content or page structure

How Data Is Used & Transmitted

All recorded interaction data is transmitted locally via the browser's postMessage API to the Test Recorder Control window running on the same device. No data is sent to any external server, third-party service, or cloud platform.

Data exists only within the active browser session and is used solely to compile reusable automated test scripts for internal QA purposes.

Permissions Explained

activeTab

Access the active tab to inject the recorder and capture user interactions during a test recording session.

scripting

Inject the test recorder script into the target tab when a recording session is initiated by the Control window.

storage

Persist recording session state (e.g. active/inactive) across tab navigations. No personal data is stored.

host_permissions

The internal app may be served on varying internal hostnames via VPN. Broad host permission ensures the extension activates regardless of the internal URL. No data is collected from public websites.

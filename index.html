Overview

FiTest Extension is an internal QA automation tool developed by Ihsan Solusi Informatika. It enables authorized testers to record user interactions within the company's internal web application — the FiTest platform — generating XPath-based test scripts by capturing clicks, inputs, and file upload events during an active recording session.

This extension is purpose-built and tightly coupled to the FiTest application. It does not operate on, interact with, or collect data from any other website or web application. The recorder only activates when explicitly triggered by the FiTest control window via the browser's postMessage API — it is functionally dormant on all other pages.

This extension is exclusively designed for internal use with the FiTest platform and is inaccessible without a valid corporate VPN connection. It is not intended for general public use and cannot function as a general-purpose browser tool.

Data We Collect

The extension collects the following data only during an active recording session:

Mouse clicks & hover position

Captured to generate XPath selectors for interacted elements.

Keyboard input values

Captured on input, textarea, and select elements to record form interactions.

Authentication information

Login credentials entered into the internal web application may be captured as part of input recording. This data is transmitted only locally via postMessage to the Control window on the same device — never stored persistently or sent to external servers.

File upload events

File name, type, and base64 content captured for test replay purposes.

XPath selectors

Generated from DOM structure to uniquely identify interacted elements.

How Data Is Used & Transmitted

All recorded interaction data is transmitted locally via the browser's postMessage API to the Test Recorder Control window running on the same device. No data is sent to any external server, third-party service, or cloud platform.

Data exists only within the active browser session and is used solely to compile reusable automated test scripts for internal QA purposes. No data is ever persisted — the extension uses no local storage, session storage, or any other persistence mechanism.

Permissions Explained

This extension requests the minimum permissions necessary to fulfil its single purpose. No activeTab, scripting, or storage permissions are declared — none are required by the extension's design.

<all_urls>

Required as a host permission because the FiTest internal application may be served under varying internal hostnames depending on the team's VPN and network configuration — these URLs cannot be predetermined or hardcoded. The content script must be present on any page that could be the recording target.

Important safeguard: Although the content script is injected on all pages, the recorder logic is entirely dormant unless the FiTest control window explicitly activates it via a postMessage signal. No data is recorded, transmitted, or processed on any page outside an active FiTest recording session.

Scope of Operation

FiTest Extension is exclusively designed to work with the FiTest platform — the company's internal QA automation application. It has no functionality outside of this context.

Active on

The FiTest internal web application, accessible via authorized corporate VPN only, when a recording session has been explicitly started by the tester.

Inactive on

Any public website, third-party web application, or any URL outside an active FiTest session — no data is recorded, processed, or transmitted in these contexts.

The extension's recording logic, control UI, and data transmission are only initialized through a deliberate action within the FiTest application. There is no passive background activity on other tabs or websites. The extension stores no data whatsoever — all in-session state lives only in memory and is discarded when the browser tab is closed or the recording session ends.
